# Stoxly Roadmap

## Vision

Stoxly aims to become a modern stock portfolio management platform that helps users track investments, analyze performance, and simulate trading using real-time market data.

The roadmap is divided into phases to ensure steady and focused development.

---

# Phase 1 — MVP (Core Platform)

Goal: Build a functional portfolio tracking platform.

## Authentication

- User registration
- User login
- JWT-based authentication
- Secure password hashing

---

## Portfolio Management

Users should be able to:

- create portfolios
- view portfolio summary
- manage multiple portfolios

---

## Stock Search

Users can search for stocks using:

- symbol
- company name

Stock data should include:

- symbol
- company name
- current price

---

## Trade Simulation

Users can simulate trades:

Buy stock
Sell stock

Trades should:

- update portfolio holdings
- update average price
- create transaction history

---

## Holdings View

Users should see:

- stock symbol
- quantity owned
- average buy price
- current price
- total value
- profit/loss

---

## Watchlist

Users can:

- add stocks to watchlist
- remove stocks
- track prices in real-time

---

## Real-Time Price Updates

Using SignalR, users should receive:

- stock price updates
- portfolio value updates

---

# Phase 2 — Portfolio Analytics

Goal: Provide deeper portfolio insights.

Features:

- portfolio value chart
- profit/loss visualization
- performance history
- asset allocation charts

---

# Phase 3 — Advanced Features

Goal: Make Stoxly feel like a full fintech product.

Features:

- dividend tracking
- trade performance analytics
- gain/loss reports
- portfolio comparison

---

# Phase 4 — Platform Enhancements

Future improvements:

- mobile application
- advanced charting
- portfolio import
- multiple market support

---

# Development Priorities

The recommended build order:

1. Authentication
2. Database schema
3. Portfolio management
4. Stock search
5. Trade simulation
6. Watchlist
7. Real-time updates
8. Analytics

---

# Long-Term Vision

Stoxly should demonstrate:

- modern full-stack architecture
- scalable backend design
- real-time systems
- production-ready engineering practices

The project should serve as a **portfolio-quality demonstration of full-stack engineering skills**.
